Here are some concrete benchmarks of web frameworks built with specific languages, set to do a series of common tasks like database retrieval, sorting, security, and encoding. It's not a perfect or rigorous comparison because it's doing a very specific set of tasks, but what it is testing is practical in its application.

As you can see, lower level, compiled languages with no garbage collector like C and Rust tend to dominate the standings for performance. Interpreted languages like javascript are not as fast, but they still remain competitive because of the excellent tooling and ecosystem javascript enjoys from its popularity.
